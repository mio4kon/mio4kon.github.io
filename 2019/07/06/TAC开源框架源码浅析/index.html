TAC开源框架源码浅析

TAC

Posted by Mio4kon on 2019-07-06

TAC相关介绍

https://github.com/alibaba/tac

  • TAC (Tiny API Cloud ) 是与 tangram 配套的服务端解决方案。当然也支持脱离 tangram 使用；
  • TAC 不是平台，也不是框架，而是一种开发模式；

下面主要是 mark 一些源码相关的关键内容

生成实例

当点击发布正式环境时,会先触发RedisMsSubscriber 然后发通知到 DefaultMsEventHandlers处理

TacInstanceContainerService 会去读取正在的容器实例:

主要做了下面几件事:

  1. 判断 cache 中是否由 Instance
  2. 无实例的话,通过TacInstanceLoadService加载实例然后返回
  3. 当要加载的版本与 cache 不一致则重新加载
  4. 将实例信息推到 redis
  5. 销毁老的实例

加载实例具体实现:

具体做了下面几件事:

  1. 通过实例 id 获取二进制
  2. 存储到本地
  3. 使用CustomerClassLoader去 load 具体实现的 TacHandler
  4. 生成TacHandler实例
  5. 如果tacHandler也实现了InitializingHandler接口,则执行afterPropertiesSet方法
  6. 所有内容放在tacInstanceInfo中返回

运行实例

控制台测试两种方式: 预发, 线上

预发环境会通过 readis 获取具体的二进制文件,然后生成具体的实例,然后运行.

TacPublishTestService#prePublishTest()

正式环境是通过 http 请求,由DefaultTacEngineService接收处理

TacPublishTestService#onlinePublishTest()

DefaultTacEngineService.class
